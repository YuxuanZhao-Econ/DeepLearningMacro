# Deep Learning Methods for Dynamic Macro Models

This repository contains Julia notebooks for studying deep-learning solution methods for dynamic macroeconomic models.

The project is research-oriented rather than a production toolkit. The current workflow starts from a small RBC model to understand the Euler-equation deep learning method in Maliar, Maliar, and Winant (2021), and then builds toward the Krusell-Smith (1998) heterogeneous-agent model.

## Requirements

The notebooks are written in Julia. The project environment is recorded in:

- `Project.toml`
- `Manifest.toml`

From the repository root, activate and instantiate the environment with:


- `Flux`

`Plots` is included in the project environment for diagnostics such as training losses, Euler-error histograms, and policy-function plots.

## Project Structure

- `RBC.ipynb`
  - warm-up notebook for the Maliar-Maliar-Winant Euler-equation method
  - solves a stochastic RBC/Brock-Mirman benchmark with a neural-network policy rule
  - uses the log-utility, full-depreciation case so the analytical policy can be plotted against the learned policy
  - converts the Euler equation into a squared-residual loss
  - trains on simulated state points using `Flux.gradient` and an Adam update
  - reports training loss, out-of-sample Euler residuals, and learned policy functions

- `Project.toml`
  - Julia project dependencies for this repository

- `Manifest.toml`
  - pinned Julia dependency versions generated by `Pkg`

- `reference/`
  - baseline paper and reference implementation
  - `JME2021.pdf` is Maliar, Maliar, and Winant (2021)
  - `deep-learning-euler-method-krusell-smith-main/` contains the TensorFlow reference code for the Krusell-Smith Euler-equation method

## Method

The main object of study is the deep learning solution framework in Maliar, Maliar, and Winant (2021). The paper shows how to solve dynamic economic models by writing the model's equilibrium conditions as nonlinear regression-style objectives.

The current notebook focuses on the Euler-equation version of the method. For the RBC model, the representative household satisfies:

```math
1
=
\beta E_t\left[
\left(\frac{c_{t+1}}{c_t}\right)^{-\gamma}
\left(\alpha e^{z_{t+1}}k_{t+1}^{\alpha-1}+1-\delta\right)
\right].
```

The notebook parameterizes the policy rule with a small neural network:

```math
\xi_t=\xi(k_t,z_t;\theta),
```

where `xi` is the consumption share out of total resources. Consumption and next-period capital are then:

```math
c_t=\xi(k_t,z_t;\theta)\left(e^{z_t}k_t^\alpha+(1-\delta)k_t\right),
```

```math
k_{t+1}=
\left[1-\xi(k_t,z_t;\theta)\right]
\left(e^{z_t}k_t^\alpha+(1-\delta)k_t\right).
```

This construction enforces feasibility and positivity by design. The training objective is the average squared Euler residual on simulated states:

```math
\mathcal L(\theta)=E\left[R(k,z,\epsilon';\theta)^2\right].
```

This mirrors the logic of the reference TensorFlow code for Krusell-Smith, but in a smaller model where every moving part can be inspected directly.

## Current Workflow

The learning plan has two stages.

1. RBC warm-up.
   Build the full deep-learning Euler method in a small representative-agent model. The goal is to understand the mapping from model equations to neural-network policy rules, residuals, simulated training points, and Euler-error diagnostics.

2. Krusell-Smith extension.
   Replace the scalar RBC state with a simulated cross section of individual wealth and productivity:

```math
S_t=(w_t^1,\ldots,w_t^N,y_t^1,\ldots,y_t^N,z_t).
```

The neural network will output individual consumption shares and, when borrowing constraints are active, multipliers for complementarity conditions. The KS1998 loss will combine Euler residuals and constraint residuals.

## Interpretation

The RBC notebook is not intended to be the final solver. It is a controlled environment for learning what the Maliar-Maliar-Winant method is doing:

- policy functions are approximated by neural networks;
- model equations become a loss function;
- training data are simulated by the model itself;
- expectations are approximated by Monte Carlo draws;
- accuracy is checked through Euler-equation residuals rather than supervised-learning prediction errors.

The next substantive step is to replace the hand-written neural-network parameterization with a standard `Flux.Chain` implementation. After that, the same Euler-residual structure can be scaled toward the Krusell-Smith model in the reference TensorFlow notebook.
